## 核心更新：防检测与防限制系统

### 一、账号安全级别系统（解决50条限制问题）

根据2025-2026年WhatsApp风控策略，新增三级账号安全级别：

| 级别         | 适用账号  | 每日上限 | 最小间隔 | 每批条数 | 批间暂停 |
| ------------ | --------- | -------- | -------- | -------- | -------- |
| **新账号**   | 注册<30天 | 30条     | 60秒     | 5条      | 3分钟    |
| **稳定账号** | 30-90天   | 80条     | 30秒     | 10条     | 2分钟    |
| **老账号**   | >90天     | 150条    | 20秒     | 15条     | 1分钟    |

**功能说明：**

- 每次发送前自动检查当日配额
- 达到上限自动停止并提示等待次日
- 实时显示"今日已发: X/上限"
- 分批发送，每批后强制暂停

### 二、模拟人类打字输入

通过Puppeteer在WhatsApp Web输入框中逐字输入：

- **按键延迟**：50-200ms 随机延迟
- **模拟打错字**：3%概率输入相邻键位错误，然后删除纠正
- **随机暂停**：15%概率暂停300-800ms（模拟思考）
- **自动聚焦**：自动找到并聚焦WhatsApp Web输入框

### 三、模拟鼠标移动

使用三次贝塞尔曲线模拟人类鼠标轨迹：

- **移动步数**：15-35步
- **曲线控制点**：添加随机偏移模拟人类不精确性
- **每步延迟**：3-11ms
- **轨迹**：非直线，带自然弯曲

### 四、浏览器指纹伪装增强

注入10项反检测脚本：

1. 移除 `navigator.webdriver` 自动化标志
2. 模拟真实插件列表
3. 覆盖语言设置为英文
4. 伪装硬件并发数为4核
5. 伪装设备内存为8GB
6. 添加 `window.chrome` 对象
7. 删除原型链上的webdriver
8. 模拟通知权限为default
9. 覆盖权限查询API
10. 追踪鼠标位置

### 五、消息长度随机化（7种策略）

| 策略               | 说明                             | 概率   |
| ------------------ | -------------------------------- | ------ |
| 插入可选短语       | "By the way"、"Also"、"FYI"等    | 6%-15% |
| 填充词替换         | very ↔ really, great ↔ wonderful | 10%    |
| 问候语后加称呼     | "Hi {name}"、"Hey there"         | 20%    |
| 零宽字符插入       | \u200B、\u200C、\u200D、\uFEFF   | 40%    |
| 标点符号后额外空格 | 改变字符数                       | 30%    |
| 行内换行           | 模拟手机输入习惯                 | 15%    |
| 标点变体           | 逗号变省略号、分号变破折号       | 10%    |

### 六、发送时间模拟人类作息

- **工作时间**（工作日 9:00-12:00, 14:00-18:00）：发送速度稍快
- **午休时间**（12:00-14:00）：发送稍慢
- **晚上**（18:00-22:00）：发送最慢
- **周末**：整体更慢
- **随机人类休息**：5%概率暂停10-40秒

### 七、消息内容随机化

- 英文问候语去重（Hi/Hello/Hey等轮换）
- 友好表情符号随机添加（20%概率）
- 英文同义词替换（project→venture/collection→range等）
- 英文句式变换
- 随机结尾语（15%概率）
- 英文标点随机化

## 技术架构

### 文件变更

| 文件                | 变更类型 | 说明                       |
| ------------------- | -------- | -------------------------- |
| `web.js`            | 大幅修改 | 核心服务端，添加防限制逻辑 |
| `public/index.html` | 大幅修改 | 前端界面，添加控制选项     |

### 新增API端点

- `GET /api/daily-stats` - 获取每日发送统计
- `POST /api/set-account-level` - 设置账号安全级别

### 核心函数

- `canSend()` - 检查是否可以发送
- `recordSend()` - 记录发送成功/失败
- `simulateHumanTyping()` - 模拟人类打字
- `simulateHumanMouseMove()` - 模拟鼠标移动
- `simulateHumanScroll()` - 模拟页面滚动
- `simulatePreSendBehavior()` - 发送前完整行为链
- `injectAntiDetectionScripts()` - 注入反检测脚本

## 使用建议

### 新账号（默认）

1. 每天最多发送30条
2. 建议分多天完成大批量发送
3. 优先发送给已有聊天记录的联系人
4. 避免在短时间内连续发送

### 账号升级策略

1. **第1-2周**：使用"新账号"级别，每天20-30条
2. **第3-4周**：切换到"稳定账号"，逐渐增加到50-80条
3. **第2个月后**：切换到"老账号"，可发送100-150条

### 最佳实践

1. 启用"消息随机化"和"消息长度随机化"
2. 使用多消息模板（3-5条随机选择）
3. 在合理时间发送（9:00-22:00）
4. 避免发送完全相同的重复内容
5. 保持稳定的网络环境

## 已知限制

2. 模拟打字功能需要headless:false才能看到效果
3. 鼠标移动模拟在后台运行时不可见
